# 类加载机制与双亲委派

## 问题

"Java 的类加载过程是怎样的？双亲委派模型的流程和意义？什么场景需要打破双亲委派？Android 的 PathClassLoader 和 DexClassLoader 有什么区别？热修复是怎么利用类加载实现的？"

## 核心答案（30 秒版本）

类加载经历**加载 → 验证 → 准备 → 解析 → 初始化**五个阶段。双亲委派模型要求类加载请求先委托给父加载器，父加载器无法完成时才自己加载，保证了核心类库的安全性和唯一性。打破双亲委派的场景：**SPI（线程上下文类加载器）、热修复（优先加载补丁 dex）、插件化（自定义 ClassLoader 加载未安装 APK）**。Android 中 PathClassLoader 加载已安装 APK，DexClassLoader 可加载任意路径的 dex/jar/apk。热修复核心原理是将补丁 dex 插入 DexPathList 数组前面，利用类加载的"首次加载即缓存"特性覆盖有 bug 的类。

## 深入解析

### 原理层（源码级）

**1. 类加载的五个阶段**

```
加载(Loading) → 验证(Verification) → 准备(Preparation) → 解析(Resolution) → 初始化(Initialization)
     │                    │                  │                 │                  │
 读取字节码          校验格式/语义      为静态变量分配内存    符号引用→直接引用    执行<clinit>
 生成Class对象       确保安全性         赋默认零值                              执行static块
```

```java
// 加载阶段：通过全限定名获取二进制字节流
// 可以从：class 文件、jar/dex、网络、动态生成（CGLib）

// 准备阶段：静态变量赋默认值
static int value = 123;
// 准备阶段：value = 0（默认值）
// 初始化阶段：value = 123（执行 <clinit>）

// 特例：static final 常量在准备阶段就赋值
static final int CONST = 123;  // 准备阶段即为 123
```

**2. 双亲委派模型**

```
                ┌─────────────────────┐
                │ Bootstrap ClassLoader │  加载 rt.jar (java.lang.*)
                │   (C++ 实现，null)    │
                └──────────┬──────────┘
                           │ 委派
                ┌──────────▼──────────┐
                │ Extension ClassLoader │  加载 ext/*.jar
                │  (ExtClassLoader)    │
                └──────────┬──────────┘
                           │ 委派
                ┌──────────▼──────────┐
                │ Application ClassLoader│  加载 classpath
                │  (AppClassLoader)     │
                └──────────┬──────────┘
                           │ 委派
                ┌──────────▼──────────┐
                │ Custom ClassLoader    │  自定义加载逻辑
                └───────────────────────┘
```

**3. ClassLoader.loadClass 源码**

```java
protected Class<?> loadClass(String name, boolean resolve) throws ClassNotFoundException {
    synchronized (getClassLoadingLock(name)) {
        // 1. 检查是否已加载
        Class<?> c = findLoadedClass(name);
        if (c == null) {
            try {
                // 2. 委派给父加载器
                if (parent != null) {
                    c = parent.loadClass(name, false);
                } else {
                    // 父为 null 说明到了 Bootstrap
                    c = findBootstrapClassOrNull(name);
                }
            } catch (ClassNotFoundException e) {
                // 父加载器无法加载
            }
            if (c == null) {
                // 3. 父加载器无法加载，自己加载
                c = findClass(name);  // 子类重写此方法
            }
        }
        if (resolve) {
            resolveClass(c);
        }
        return c;
    }
}
```

**5. DexPathList 核心结构**

```java
// DexPathList.java
final class DexPathList {
    private Element[] dexElements;  // dex 文件数组，按顺序查找

    public Class<?> findClass(String name, List<Throwable> suppressed) {
        // 遍历 dexElements，找到即返回（先找到的优先）
        for (Element element : dexElements) {
            Class<?> clazz = element.findClass(name, definingContext, suppressed);
            if (clazz != null) {
                return clazz;  // 第一个匹配的 dex 中的类被加载
            }
        }
        return null;
    }
}
```

### 实战层（项目经验）

**1. 热修复原理（Tinker/QFix 方案）**

```java
// 核心思路：将补丁 dex 插入 dexElements 数组最前面
public static void hotfix(Context context, String patchDexPath) {
    try {
        // 1. 获取当前 ClassLoader 的 pathList
        ClassLoader classLoader = context.getClassLoader();
        Object pathList = getField(classLoader, "pathList");
        Object[] currentElements = (Object[]) getField(pathList, "dexElements");

        // 2. 加载补丁 dex，获取其 dexElements
        DexClassLoader patchLoader = new DexClassLoader(
            patchDexPath, context.getCacheDir().getPath(), null, classLoader);
        Object patchPathList = getField(patchLoader, "pathList");
        Object[] patchElements = (Object[]) getField(patchPathList, "dexElements");

        // 3. 合并：补丁在前，原始在后
        Object[] newElements = (Object[]) Array.newInstance(
            currentElements.getClass().getComponentType(),
            patchElements.length + currentElements.length);
        System.arraycopy(patchElements, 0, newElements, 0, patchElements.length);
        System.arraycopy(currentElements, 0, newElements, patchElements.length,
            currentElements.length);

        // 4. 替换 pathList.dexElements
        setField(pathList, "dexElements", newElements);
    } catch (Exception e) {
        e.printStackTrace();
    }
}
```

**2. 插件化加载未安装 APK**

```java
// 加载插件 APK 中的类
public class PluginManager {
    private DexClassLoader pluginClassLoader;

    public void loadPlugin(String apkPath) {
        pluginClassLoader = new DexClassLoader(
            apkPath,
            context.getDir("plugin_opt", 0).getAbsolutePath(),
            null,  // native lib path
            context.getClassLoader()  // 父加载器
        );
    }

    public Class<?> loadPluginClass(String className) throws ClassNotFoundException {
        return pluginClassLoader.loadClass(className);
    }
}
```
