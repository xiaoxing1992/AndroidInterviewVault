# 反射机制与动态代理

## 问题

"反射的实现原理是什么？Method.invoke 内部做了什么？JDK 动态代理和 CGLIB 有什么区别？Retrofit 是怎么用动态代理把接口变成网络请求的？反射性能差，怎么优化？"

## 核心答案（30 秒版本）

反射通过 **Class 对象 + 元数据**（Method/Field/Constructor）在运行时获取类型信息并操作对象。`Method.invoke` 经过权限检查、参数装箱、本地方法调用三步，前 16 次走解释执行（NativeMethodAccessor），之后生成字节码（GeneratedMethodAccessor）走 JIT。**JDK 动态代理基于接口**（Proxy.newProxyInstance + InvocationHandler），**CGLIB 基于继承**（ASM 字节码生成子类）。Retrofit 用 JDK 动态代理拦截接口方法调用，解析注解后转换为 OkHttp 请求。性能优化：缓存 Method、setAccessible(true) 跳过权限检查、用 MethodHandle 替代 Method。

## 深入解析

### 原理层（源码级）

**1. Class 对象的获取与结构**

```java
// 三种获取方式
Class<User> clazz1 = User.class;                      // 编译期静态获取
Class<?> clazz2 = user.getClass();                    // 运行时实例获取
Class<?> clazz3 = Class.forName("com.example.User");  // 触发类加载

// Class 对象包含的元数据
clazz.getDeclaredFields();      // 所有字段（含 private）
clazz.getDeclaredMethods();     // 所有方法
clazz.getDeclaredConstructors();
clazz.getAnnotations();
clazz.getGenericSuperclass();   // 带泛型的父类
```

**2. Method.invoke 源码分析**

```java
// java.lang.reflect.Method
public Object invoke(Object obj, Object... args) {
    // 1. 权限检查（除非 setAccessible(true)）
    if (!override) {
        Class<?> caller = Reflection.getCallerClass();
        checkAccess(caller, clazz, obj, modifiers);
    }
    // 2. 通过 MethodAccessor 实际执行
    MethodAccessor ma = methodAccessor;
    if (ma == null) {
        ma = acquireMethodAccessor();  // 懒加载
    }
    return ma.invoke(obj, args);
}

// MethodAccessor 实现切换策略
class DelegatingMethodAccessorImpl extends MethodAccessorImpl {
    private MethodAccessorImpl delegate;

    public Object invoke(Object obj, Object[] args) {
        return delegate.invoke(obj, args);
    }
}

// 默认 NativeMethodAccessorImpl，调用次数超过阈值（默认 15）
// 切换为 GeneratedMethodAccessor，由 ASM 生成具体调用字节码
// 阈值由 sun.reflect.inflationThreshold 控制
```

**3. JDK 动态代理实现**

```java
// 核心 API
Object proxy = Proxy.newProxyInstance(
    classLoader,            // 类加载器
    new Class[]{Foo.class}, // 必须是接口
    handler                 // InvocationHandler
);

// 内部生成的代理类（伪代码）
public final class $Proxy0 extends Proxy implements Foo {
    private static Method m1; // hashCode
    private static Method m2; // equals
    private static Method m3; // doSomething

    public final void doSomething(String arg) {
        try {
            super.h.invoke(this, m3, new Object[]{arg});
        } catch (Throwable t) { /* ... */ }
    }

    static {
        m3 = Class.forName("Foo").getMethod("doSomething", String.class);
    }
}
```

**4. CGLIB 实现原理**

```java
// CGLIB 通过 ASM 生成目标类的子类
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(User.class);  // 继承目标类
enhancer.setCallback(new MethodInterceptor() {
    @Override
    public Object intercept(Object obj, Method method,
                             Object[] args, MethodProxy proxy) throws Throwable {
        return proxy.invokeSuper(obj, args);  // FastClass 索引调用，比反射快
    }
});
User proxy = (User) enhancer.create();

// CGLIB vs JDK 动态代理对比：
// JDK: 基于接口，必须实现接口；调用通过反射 invoke
// CGLIB: 基于继承，无需接口；FastClass 用索引代替反射，性能更高
//        但不能代理 final 类/方法
```

**5. Retrofit 动态代理核心**

```java
// Retrofit.create() 源码精简版
public <T> T create(final Class<T> service) {
    return (T) Proxy.newProxyInstance(
        service.getClassLoader(),
        new Class<?>[]{service},
        new InvocationHandler() {
            @Override
            public Object invoke(Object proxy, Method method, Object[] args) {
                // 1. 解析方法注解，缓存为 ServiceMethod
                ServiceMethod<Object> serviceMethod =
                    (ServiceMethod<Object>) loadServiceMethod(method);
                // 2. 转换为 HTTP 调用
                return serviceMethod.invoke(args);
            }
        });
}

// ServiceMethod 缓存避免重复解析
private final Map<Method, ServiceMethod<?>> serviceMethodCache = new ConcurrentHashMap<>();

ServiceMethod<?> loadServiceMethod(Method method) {
    ServiceMethod<?> result = serviceMethodCache.get(method);
    if (result != null) return result;
    synchronized (serviceMethodCache) {
        result = serviceMethodCache.get(method);
        if (result == null) {
            result = ServiceMethod.parseAnnotations(this, method);  // 解析 @GET/@Query 等
            serviceMethodCache.put(method, result);
        }
    }
    return result;
}
```

### 实战层（项目经验）

**1. 反射性能优化**

```java
// ❌ 慢：每次都查找 Method
public void slow(Object obj) throws Exception {
    Method method = obj.getClass().getDeclaredMethod("doWork");
    method.invoke(obj);
}

// ✅ 快：缓存 Method + setAccessible
public class FastReflector {
    private static final ConcurrentHashMap<String, Method> CACHE = new ConcurrentHashMap<>();

    public static Object invoke(Object obj, String methodName) throws Exception {
        String key = obj.getClass().getName() + "#" + methodName;
        Method method = CACHE.computeIfAbsent(key, k -> {
            try {
                Method m = obj.getClass().getDeclaredMethod(methodName);
                m.setAccessible(true);  // 跳过 access check
                return m;
            } catch (NoSuchMethodException e) {
                throw new RuntimeException(e);
            }
        });
        return method.invoke(obj);
    }
}

// ✅ 更快：MethodHandle (Java 7+)
MethodHandles.Lookup lookup = MethodHandles.lookup();
MethodHandle handle = lookup.findVirtual(User.class, "getName",
    MethodType.methodType(String.class));
String name = (String) handle.invoke(user);
```

**3. 反射性能基准（参考数据）**

```
1000 万次方法调用：
直接调用:        ~10ms
Method.invoke:   ~150ms (15x)
Method.invoke + setAccessible(true): ~80ms
MethodHandle:    ~30ms (3x)
CGLIB FastClass: ~25ms (2.5x)
```

**4. Android Hidden API 限制（P+）**

```java
// Android 9+ 限制非 SDK 接口反射
// 解决方案：通过 setHiddenApiExemptions 豁免
Method forName = Class.class.getDeclaredMethod("forName", String.class);
Method getDeclaredMethod = Class.class.getDeclaredMethod("getDeclaredMethod",
    String.class, Class[].class);

Class<?> vmRuntimeClass = (Class<?>) forName.invoke(null, "dalvik.system.VMRuntime");
Method getRuntime = (Method) getDeclaredMethod.invoke(vmRuntimeClass, "getRuntime", null);
Method setExemptions = (Method) getDeclaredMethod.invoke(vmRuntimeClass,
    "setHiddenApiExemptions", new Class[]{String[].class});

Object vmRuntime = getRuntime.invoke(null);
setExemptions.invoke(vmRuntime, new Object[]{new String[]{"L"}});  // 豁免所有
```
