# 启动优化-耗时分析与优化手段

## 问题

> 你们项目的冷启动时间是多少？怎么度量的？做了哪些优化能把启动时间降下来？

## 核心答案（30秒版本）

启动耗时通过 Perfetto/Systrace 进行精确度量，关注从 `Process.start` 到首帧绘制完成（TTID/TTFD）的全链路。优化手段分四层：任务重排（异步初始化 + 拓扑排序）、延迟加载（按需初始化非关键组件）、减少主线程阻塞（避免 ContentProvider 滥用）、Baseline Profile（AOT 编译热路径）。核心思路是把串行变并行、把同步变异步、把启动时变使用时。

## 深入解析

### 原理层（源码级）

**冷启动全流程：**

```
Zygote fork → Application.attachBaseContext → ContentProvider.onCreate (按 priority 排序)
→ Application.onCreate → Activity.onCreate → onStart → onResume → 首帧绘制 (ViewRootImpl.performTraversals)
```

**TTID vs TTFD：**
- TTID (Time To Initial Display)：系统自动报告，首帧绘制完成时间
- TTFD (Time To Full Display)：需手动调用 `reportFullyDrawn()`，表示有意义内容展示完成

**Perfetto 度量关键 Trace：**

```kotlin
// 系统自动插入的 trace 点

```

**自定义精确度量：**

```kotlin
class MyApp : Application() {
    override fun attachBaseContext(base: Context) {
        super.attachBaseContext(base)
        // 最早的时机记录启动时间
        LaunchTimer.startRecord()
    }
}

// 首页首帧回调
class MainActivity : AppCompatActivity() {
    override fun onWindowFocusChanged(hasFocus: Boolean) {
        super.onWindowFocusChanged(hasFocus)
        if (hasFocus) {
            LaunchTimer.endRecord() // TTID
        }
    }
}
```

**异步初始化框架核心原理（拓扑排序 + 有向无环图）：**

```kotlin
// 启动任务抽象
abstract class StartupTask {
    abstract fun run()
    open fun dependencies(): List<Class<out StartupTask>> = emptyList()
    open fun runOnMainThread(): Boolean = false
}

// 拓扑排序调度
class TaskDispatcher {
    private val taskMap = mutableMapOf<Class<*>, StartupTask>()
    private val inDegreeMap = mutableMapOf<Class<*>, Int>()

    fun dispatch() {
        // Kahn 算法拓扑排序
        val queue: Queue<StartupTask> = LinkedList()
        inDegreeMap.filter { it.value == 0 }.forEach { (clazz, _) ->
            queue.offer(taskMap[clazz])
        }

        val mainThreadTasks = mutableListOf<StartupTask>()
        val asyncTasks = mutableListOf<StartupTask>()

        while (queue.isNotEmpty()) {
            val task = queue.poll()
            if (task.runOnMainThread()) {
                mainThreadTasks.add(task)
            } else {
                asyncTasks.add(task)
            }
            // 更新入度，将新的零入度任务入队
        }

        // 异步任务提交线程池
        asyncTasks.forEach { threadPool.execute(it::run) }
        // 主线程任务顺序执行
        mainThreadTasks.forEach { it.run() }
        // CountDownLatch 等待关键任务完成
    }
}
```

**Baseline Profile 原理：**

Baseline Profile 本质是将热路径方法列表提供给 ART，在安装时通过 `dex2oat` 进行 AOT 编译，避免运行时 JIT 编译的开销。

```
// baseline-prof.txt 格式示例
HSPLcom/example/MyApp;->onCreate()V
HSPLcom/example/MainActivity;->onCreate(Landroid/os/Bundle;)V
Lcom/example/repo/DataRepository;->loadData()Ljava/util/List;
```

- `H` = Hot（热方法，AOT 编译）
- `S` = Startup（启动路径方法）
- `P` = Post-startup（启动后方法）
- `L` = 仅预加载类

### 实战层（项目经验）

**优化前后对比（某电商 App）：**

| 阶段 | 优化前 | 优化后 | 手段 |
|------|--------|--------|------|
| Application.onCreate | 1200ms | 350ms | 异步初始化 + 延迟加载 |
| ContentProvider | 400ms | 50ms | 合并为 1 个 + 移除不必要的 |
| 首页 Activity | 600ms | 200ms | ViewStub + 预加载数据 |
| 首帧渲染 | 300ms | 150ms | Baseline Profile + 布局优化 |
| **总计 TTID** | **2500ms** | **750ms** | - |

**关键实战技巧：**

1. **ContentProvider 治理**：第三方 SDK 滥用 ContentProvider 自动初始化，用 `tools:node="remove"` 移除后手动延迟初始化

```xml
<provider
    android:name="com.thirdparty.sdk.InitProvider"
    android:authorities="${applicationId}.thirdparty-init"
    tools:node="remove" />
```

2. **闪屏页利用**：在 SplashActivity 的 windowBackground 设置品牌图，利用这段时间做异步初始化

```xml
<style name="SplashTheme" parent="Theme.AppCompat.NoActionBar">
    <item name="android:windowBackground">@drawable/splash_background</item>
</style>
```

3. **Baseline Profile 生成**：

```kotlin
@RunWith(AndroidJUnit4::class)
class BaselineProfileGenerator {
    @get:Rule
    val rule = BaselineProfileRule()

    @Test
    fun generateBaselineProfile() {
        rule.collect(packageName = "com.example.app") {
            // 模拟启动路径
            startActivityAndWait()
            device.findObject(By.text("首页")).click()
            device.waitForIdle()
        }
    }
}
```

4. **线上监控方案**：

```kotlin
object LaunchTimer {
    private var startTime = 0L
    private var isFirstLaunch = true

    fun startRecord() {
        if (isFirstLaunch) {
            startTime = SystemClock.elapsedRealtime()
        }
    }

    fun endRecord(tag: String = "TTID") {
        if (isFirstLaunch) {
            val cost = SystemClock.elapsedRealtime() - startTime
            // 上报到 APM 平台
            ApmReporter.report("launch_$tag", cost)
            isFirstLaunch = false
        }
    }
}
```

启动优化 = 度量先行（Perfetto）+ 任务并行（拓扑排序）+ 延迟加载 + AOT 预编译（Baseline Profile），核心是把串行变并行、同步变异步。
