# 包体积优化-R8与资源压缩

## 问题

> R8 和 ProGuard 有什么区别？包体积优化你们做了哪些事情？效果怎么样？

## 核心答案（30秒版本）

R8 是 Google 开发的 ProGuard 替代品，集成了脱糖（Desugaring）+ 压缩（Shrinking）+ 混淆（Obfuscation）+ 优化（Optimization）为一体，编译速度更快、产物更小。包体积优化从代码（R8 Tree Shaking、移除无用代码）、资源（shrinkResources、WebP 转换、语言精简）、So 库（ABI 过滤、动态下发）、Dex（多 Dex 优化）四个维度入手。

## 深入解析

### 原理层（源码级）

**R8 vs ProGuard 架构对比：**

```
ProGuard 流程（多步骤）：
.java → javac → .class → ProGuard(shrink+optimize+obfuscate) → dx → .dex

R8 流程（单步骤）：
.java → javac → .class → R8(desugar+shrink+optimize+obfuscate+dex) → .dex

关键区别：
1. R8 直接输出 dex，省去 dx/d8 步骤
2. R8 内置 desugar，支持 Java 8+ 语法
3. R8 优化更激进（方法内联、常量折叠、枚举优化）
4. R8 与 AGP 深度集成，理解 Android 特有语义
```

**R8 Tree Shaking 原理：**

```
Entry Points (keep 规则指定)
    ↓
从入口点出发，标记所有可达的类、方法、字段
    ↓
未被标记的代码 = Dead Code → 移除
    ↓
优化：方法内联、常量传播、无用分支消除
    ↓
混淆：短名称替换（a, b, c...）
    ↓
输出精简后的 dex
```

**R8 关键优化技术：**

```java
// 1. 枚举优化 - 将枚举替换为 int 常量
// 优化前
enum class Direction { UP, DOWN, LEFT, RIGHT }
fun move(dir: Direction) = when(dir) { ... }

// R8 优化后（反编译）
// Direction 类被移除，替换为 int
fun move(dir: Int) = when(dir) { 0 -> ... 1 -> ... }

// 2. 方法内联
// 优化前
fun getLength(s: String): Int = s.length
fun process(s: String) { val len = getLength(s) }

// R8 优化后
fun process(s: String) { val len = s.length } // getLength 被内联

// 3. 类合并（Vertical Class Merging）
// 只有一个实现的接口，接口被合并到实现类中
```

**资源压缩原理（shrinkResources）：**

```groovy
// build.gradle
android {
    buildTypes {
        release {
            minifyEnabled true       // 代码压缩（R8）
            shrinkResources true     // 资源压缩
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
                'proguard-rules.pro'
        }
    }
}
```

```
资源压缩流程：
1. R8 先移除无用代码
2. ResourcesShrinker 分析代码中的资源引用
3. 通过 Resources.getIdentifier() 动态引用分析
4. 未被引用的资源替换为空文件（保留 ID 映射）
5. 配合 resConfigs 移除不需要的语言/密度资源
```

**keep 规则详解：**

```proguard
# 保留类名和所有成员
-keep class com.example.Model { *; }

# 保留类名，成员可以混淆
-keepnames class com.example.Model

# 保留类成员（如果类被保留的话）
-keepclassmembers class * implements android.os.Parcelable {
    static ** CREATOR;
}

# 保留类和成员名（不混淆），但允许 shrink
-keepnames class * extends android.app.Activity

# R8 特有：优化时保留方法签名（用于反射）
-keepattributes Signature
-keepattributes *Annotation*
```

### 实战层（项目经验）

**1. 完整包体积优化方案（某 App 从 85MB → 52MB）：**

| 优化项 | 减少体积 | 占比 |
|--------|----------|------|
| So 库 ABI 过滤（仅 arm64-v8a） | -12MB | 36% |
| 资源压缩 + WebP 转换 | -8MB | 24% |
| R8 全量混淆 + Tree Shaking | -6MB | 18% |
| 移除无用语言资源 | -3MB | 9% |
| 动态下发非核心 So | -2.5MB | 8% |
| Dex 优化 | -1.5MB | 5% |

**2. 资源优化实战：**

```groovy
android {
    defaultConfig {
        // 只保留中英文资源
        resConfigs "zh-rCN", "en"

        // 只保留 arm64
        ndk {
            abiFilters 'arm64-v8a'
        }
    }
}

// 图片压缩 Gradle Task
tasks.register("compressImages") {
    doLast {
        fileTree("src/main/res").matching {
            include "**/*.png"
            exclude "**/*.9.png" // 排除 .9 图
        }.forEach { file ->
            // PNG → WebP 转换（无损压缩可减少 25-35%）
            exec {
                commandLine "cwebp", "-q", "80", file.path,
                    "-o", file.path.replace(".png", ".webp")
            }
        }
    }
}
```

**3. So 库动态下发：**

```kotlin
// 非核心 So 库按需下载
class SoLoader {
    fun loadSoIfNeeded(soName: String, callback: (Boolean) -> Unit) {
        val soFile = File(context.filesDir, "libs/$soName.so")
        if (soFile.exists()) {
            System.load(soFile.absolutePath)
            callback(true)
        } else {
            // 从 CDN 下载
            downloadSo(soName) { success ->
                if (success) {
                    System.load(soFile.absolutePath)
                }
                callback(success)
            }
        }
    }
}
```
