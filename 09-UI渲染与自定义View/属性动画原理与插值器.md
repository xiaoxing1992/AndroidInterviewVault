# 属性动画原理与插值器

## 问题

"属性动画的底层原理是什么？ValueAnimator 和 ObjectAnimator 的区别？TypeEvaluator 和 Interpolator 各自的作用？Choreographer 是怎么驱动动画的？"

## 核心答案（30秒版本）

属性动画通过 Choreographer 注册 VSYNC 回调，每帧计算当前时间分数（fraction），经 Interpolator 变换为动画进度，再由 TypeEvaluator 计算属性当前值。ValueAnimator 只计算值需手动应用，ObjectAnimator 通过反射/setter 自动更新目标属性。Choreographer 是帧调度中枢，统一管理输入、动画、绘制三类回调的执行时机。

## 深入解析

### 原理层（源码级）

**动画驱动链路：**

```
VSYNC 信号 → Choreographer.doFrame()
  → CALLBACK_ANIMATION → AnimationHandler.doAnimationFrame()
    → ValueAnimator.doAnimationFrame()
      → animateValue()
        → Interpolator.getInterpolation(fraction) → 变换后的 fraction
        → TypeEvaluator.evaluate(fraction, startValue, endValue) → 当前值
        → 通知 AnimatorUpdateListener / 反射设置属性
```

**Choreographer 核心机制：**

```java
// Choreographer.java
public final class Choreographer {
    // 四种回调类型
    public static final int CALLBACK_INPUT = 0;      // 输入事件
    public static final int CALLBACK_ANIMATION = 1;  // 动画
    public static final int CALLBACK_INSETS_ANIMATION = 2;
    public static final int CALLBACK_TRAVERSAL = 3;  // View 绘制
    
    private final CallbackQueue[] mCallbackQueues;
    
    void doFrame(long frameTimeNanos, int frame) {
        // 按优先级依次执行
        doCallbacks(Choreographer.CALLBACK_INPUT, frameTimeNanos);
        doCallbacks(Choreographer.CALLBACK_ANIMATION, frameTimeNanos);
        doCallbacks(Choreographer.CALLBACK_INSETS_ANIMATION, frameTimeNanos);
        doCallbacks(Choreographer.CALLBACK_TRAVERSAL, frameTimeNanos);
    }
    
    // 注册下一帧回调
    public void postFrameCallback(FrameCallback callback) {
        postFrameCallbackDelayed(callback, 0);
    }
    
    private void scheduleFrameLocked(long now) {
        // 通过 native 层注册 VSYNC 信号监听
        scheduleVsyncLocked();
    }
}
```

**ValueAnimator 核心流程：**

```java
// ValueAnimator.java
public class ValueAnimator extends Animator {
    
    public void start() {
        // 注册到 AnimationHandler
        AnimationHandler.getInstance().addAnimationFrameCallback(this, delay);
    }
    
    // 每帧回调
    public final boolean doAnimationFrame(long frameTime) {
        // 计算时间分数 [0, 1]
        float fraction = (currentTime - mStartTime) / mDuration;
        fraction = clamp(fraction, 0f, 1f);
        
        animateValue(fraction);
        return fraction >= 1f; // 动画是否结束
    }
    
    void animateValue(float fraction) {
        // 1. Interpolator 变换
        fraction = mInterpolator.getInterpolation(fraction);
        
        // 2. TypeEvaluator 计算当前值
        for (PropertyValuesHolder pvh : mValues) {
            pvh.calculateValue(fraction);
        }
        
        // 3. 通知监听器
        for (AnimatorUpdateListener listener : mUpdateListeners) {
            listener.onAnimationUpdate(this);
        }
    }
}
```

**ObjectAnimator 反射机制：**

```java
// ObjectAnimator.java
public final class ObjectAnimator extends ValueAnimator {
    private Object mTarget; // 弱引用持有目标对象
    
    @Override
    void animateValue(float fraction) {
        super.animateValue(fraction);
        // 通过 PropertyValuesHolder 设置属性值
        for (PropertyValuesHolder pvh : mValues) {
            pvh.setAnimatedValue(mTarget);
        }
    }
}

// PropertyValuesHolder.java
void setAnimatedValue(Object target) {
    if (mProperty != null) {
        // Property 抽象类方式（推荐，无反射）
        mProperty.set(target, getAnimatedValue());
    } else if (mSetter != null) {
        // 反射调用 setter
        mSetter.invoke(target, getAnimatedValue());
    }
}
```

**Interpolator 接口：**

```java
public interface TimeInterpolator {
    // input: 时间分数 [0, 1]，线性递增
    // return: 动画进度 [0, 1]（可超出范围实现弹性效果）
    float getInterpolation(float input);
}

// AccelerateDecelerateInterpolator 源码
public float getInterpolation(float input) {
    return (float)(Math.cos((input + 1) * Math.PI) / 2.0f) + 0.5f;
}

// OvershootInterpolator 源码（超出目标值再回弹）
public float getInterpolation(float t) {
    t -= 1.0f;
    return t * t * ((mTension + 1) * t + mTension) + 1.0f;
}
```

**TypeEvaluator 接口：**

```java
public interface TypeEvaluator<T> {
    // fraction: Interpolator 变换后的进度
    // startValue/endValue: 动画起止值
    T evaluate(float fraction, T startValue, T endValue);
}

// IntEvaluator
public Integer evaluate(float fraction, Integer startValue, Integer endValue) {
    int startInt = startValue;
    return (int)(startInt + fraction * (endValue - startInt));
}

// ArgbEvaluator（颜色渐变，分通道插值）
public Object evaluate(float fraction, Object startValue, Object endValue) {
    int startA = (startInt >> 24) & 0xff;
    int startR = (startInt >> 16) & 0xff;
    // ... 各通道线性插值
    return (startA + (int)(fraction * (endA - startA))) << 24 | ...;
}
```

### 实战层（项目经验）

**自定义 TypeEvaluator 实现路径动画：**

```kotlin
// 贝塞尔曲线路径动画
class BezierEvaluator(private val controlPoint: PointF) : TypeEvaluator<PointF> {
    override fun evaluate(fraction: Float, start: PointF, end: PointF): PointF {
        // 二阶贝塞尔曲线公式
        val x = (1 - fraction).pow(2) * start.x +
                2 * (1 - fraction) * fraction * controlPoint.x +
                fraction.pow(2) * end.x
        val y = (1 - fraction).pow(2) * start.y +
                2 * (1 - fraction) * fraction * controlPoint.y +
                fraction.pow(2) * end.y
        return PointF(x, y)
    }
}

// 使用
val animator = ValueAnimator.ofObject(
    BezierEvaluator(PointF(300f, 0f)),
    PointF(0f, 0f),
    PointF(600f, 600f)
)
animator.addUpdateListener { animation ->
    val point = animation.animatedValue as PointF
    view.translationX = point.x
    view.translationY = point.y
}
```

**自定义 Interpolator 实现弹簧效果：**

```kotlin
class SpringInterpolator(
    private val factor: Float = 0.4f
) : TimeInterpolator {
    override fun getInterpolation(input: Float): Float {
        // 阻尼振荡公式
        return (2.0.pow(-10.0 * input) * 
                sin((input - factor / 4) * (2 * PI) / factor) + 1).toFloat()
    }
}
```

**AnimatorSet 组合动画：**

```kotlin
val scaleX = ObjectAnimator.ofFloat(view, "scaleX", 1f, 1.2f, 1f)
val scaleY = ObjectAnimator.ofFloat(view, "scaleY", 1f, 1.2f, 1f)
val alpha = ObjectAnimator.ofFloat(view, "alpha", 1f, 0.5f, 1f)

AnimatorSet().apply {
    playTogether(scaleX, scaleY) // 同时执行缩放
    play(alpha).after(scaleX)    // 缩放完成后执行透明度
    duration = 300
    interpolator = FastOutSlowInInterpolator()
    start()
}
```

**ViewPropertyAnimator（性能最优）：**

```kotlin
// 内部批量优化，减少 JNI 调用和 invalidate 次数
view.animate()
    .translationX(100f)
    .alpha(0.5f)
    .scaleX(1.2f)
    .setDuration(300)
    .setInterpolator(AccelerateDecelerateInterpolator())
    .withEndAction { /* 动画结束回调 */ }
    .start()
```

**动画性能注意事项：**

```kotlin
// 1. 避免在 onAnimationUpdate 中触发 requestLayout
animator.addUpdateListener {
    // ✗ 触发 measure + layout + draw
    view.layoutParams.width = it.animatedValue as Int
    view.requestLayout()
    
    // ✓ 只触发 draw（硬件加速下更新 RenderNode 属性）
    view.translationX = it.animatedValue as Float
}

// 2. 硬件层加速复杂动画
view.setLayerType(View.LAYER_TYPE_HARDWARE, null)
animator.addListener(object : AnimatorListenerAdapter() {
    override fun onAnimationEnd(animation: Animator) {
        view.setLayerType(View.LAYER_TYPE_NONE, null) // 动画结束移除
    }
})

// 3. 使用 RenderThread 动画（API 24+，不阻塞主线程）
val node = view.createRenderNodeAnimator(RenderNodeAnimator.TRANSLATION_X, 100f)
node.setDuration(300)
node.start()
```
