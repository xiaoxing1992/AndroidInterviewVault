# 屏幕适配方案对比

## 问题

"Android 屏幕适配有哪些方案？dp/sp 的原理是什么？今日头条适配方案的原理和优缺点？smallestWidth 限定符方案怎么用？Compose 中怎么做适配？"

## 核心答案（30秒版本）

dp 是密度无关像素，px = dp * (dpi / 160)；sp 额外受用户字体缩放影响。今日头条方案通过修改 DisplayMetrics.density 使设计稿 dp 值直接对应屏幕宽度，侵入性强但适配精准。smallestWidth 限定符方案通过生成多套 dimens.xml 覆盖不同屏幕宽度，无侵入但增加包体积。Compose 中可通过 LocalDensity 自定义 Density 实现等比缩放，或使用 WindowSizeClass 做响应式布局。

## 深入解析

### 原理层（源码级）

**dp/sp/px 转换原理：**

```java
// TypedValue.java
public static float applyDimension(int unit, float value, DisplayMetrics metrics) {
    switch (unit) {
        case COMPLEX_UNIT_PX:
            return value;
        case COMPLEX_UNIT_DIP: // dp
            return value * metrics.density;
            // density = dpi / 160
            // 160dpi 设备：1dp = 1px
            // 320dpi 设备：1dp = 2px
            // 480dpi 设备：1dp = 3px
        case COMPLEX_UNIT_SP:
            return value * metrics.scaledDensity;
            // scaledDensity = density * fontScale
            // fontScale 由系统设置中的字体大小决定
        case COMPLEX_UNIT_PT:
            return value * metrics.xdpi * (1.0f / 72);
    }
    return 0;
}
```

**DisplayMetrics 关键字段：**

```java
public class DisplayMetrics {
    public float density;        // dpi / 160，如 2.0（320dpi）
    public int densityDpi;       // 实际 dpi，如 320
    public float scaledDensity;  // density * fontScale
    public int widthPixels;      // 屏幕宽度 px
    public int heightPixels;     // 屏幕高度 px
    public float xdpi;           // X 方向精确 dpi
    public float ydpi;           // Y 方向精确 dpi
}
```

**今日头条方案原理：**

```java
// 核心思路：设计稿宽度 360dp，让所有设备的屏幕宽度都等于 360dp
// 公式：density = screenWidthPx / designWidthDp

// 例如：1080px 宽的设备
// 原始 density = 3.0（1080 / 360 = 3.0，刚好）
// 1440px 宽的设备
// 修改 density = 1440 / 360 = 4.0

public class ScreenAdaptHelper {
    private static float originalDensity;
    private static float originalScaledDensity;
    
    public static void adapt(Activity activity, float designWidthDp) {
        DisplayMetrics appMetrics = activity.getApplication().getResources().getDisplayMetrics();
        
        if (originalDensity == 0) {
            originalDensity = appMetrics.density;
            originalScaledDensity = appMetrics.scaledDensity;
        }
        
        // 计算新 density
        float targetDensity = appMetrics.widthPixels / designWidthDp;
        float targetScaledDensity = targetDensity * (originalScaledDensity / originalDensity);
        int targetDensityDpi = (int) (targetDensity * 160);
        
        // 修改 Application 的 DisplayMetrics
        appMetrics.density = targetDensity;
        appMetrics.scaledDensity = targetScaledDensity;
        appMetrics.densityDpi = targetDensityDpi;
        
        // 修改 Activity 的 DisplayMetrics
        DisplayMetrics activityMetrics = activity.getResources().getDisplayMetrics();
        activityMetrics.density = targetDensity;
        activityMetrics.scaledDensity = targetScaledDensity;
        activityMetrics.densityDpi = targetDensityDpi;
    }
    
    public static void cancel(Activity activity) {
        DisplayMetrics metrics = activity.getResources().getDisplayMetrics();
        metrics.density = originalDensity;
        metrics.scaledDensity = originalScaledDensity;
        metrics.densityDpi = (int) (originalDensity * 160);
    }
}
```

**smallestWidth 限定符方案：**

```
// 目录结构
res/
├── values/dimens.xml              (默认，360dp 基准)
├── values-sw320dp/dimens.xml      (320dp 宽设备)
├── values-sw360dp/dimens.xml      (360dp 宽设备)
├── values-sw400dp/dimens.xml      (400dp 宽设备)
├── values-sw411dp/dimens.xml      (411dp 宽设备)
├── values-sw480dp/dimens.xml      (480dp 宽设备)
└── values-sw600dp/dimens.xml      (平板)
```

```xml
<!-- values-sw360dp/dimens.xml（设计稿基准 360dp）-->
<resources>
    <dimen name="dp_1">1dp</dimen>
    <dimen name="dp_2">2dp</dimen>
    <!-- ... -->
    <dimen name="dp_360">360dp</dimen>
</resources>

<!-- values-sw400dp/dimens.xml（等比缩放）-->
<resources>
    <dimen name="dp_1">1.11dp</dimen>  <!-- 400/360 * 1 -->
    <dimen name="dp_2">2.22dp</dimen>  <!-- 400/360 * 2 -->
    <!-- ... -->
    <dimen name="dp_360">400dp</dimen>
</resources>
```

系统选择规则：设备 smallestWidth = min(widthDp, heightDp)，选择不超过该值的最大限定符目录。

### 实战层（项目经验）

**方案对比：**

| 维度 | 今日头条方案 | smallestWidth | Compose 自定义 Density |
|---|---|---|---|
| 侵入性 | 高（修改全局 density） | 低（资源限定符） | 低（CompositionLocal） |
| 精度 | 高（像素级） | 中（离散档位） | 高 |
| 包体积 | 无影响 | 增大（多套 dimens） | 无影响 |
| 第三方库兼容 | 可能影响 | 完全兼容 | 仅影响 Compose 层 |
| 字体适配 | 需额外处理 | 天然支持 | 灵活控制 |
| 折叠屏/平板 | 需额外处理 | 天然支持 | 配合 WindowSizeClass |

**今日头条方案改进版（AndroidAutoSize）：**

```kotlin
// 基于 Activity 生命周期自动适配
class AdaptActivity : AppCompatActivity() {
    override fun getResources(): Resources {
        // 在 Activity 级别修改，避免影响全局
        val resources = super.getResources()
        val metrics = resources.displayMetrics
        val targetDensity = metrics.widthPixels / 360f
        metrics.density = targetDensity
        metrics.scaledDensity = targetDensity * (Settings.System.getFloat(
            contentResolver, Settings.System.FONT_SCALE, 1f))
        metrics.densityDpi = (targetDensity * 160).toInt()
        return resources
    }
}
```

**Compose 适配方案：**

```kotlin
// 方案 1：自定义 Density 等比缩放
@Composable
fun AdaptiveApp(designWidthDp: Float = 360f, content: @Composable () -> Unit) {
    val configuration = LocalConfiguration.current
    val screenWidthDp = configuration.screenWidthDp
    
    val customDensity = Density(
        density = screenWidthDp / designWidthDp * LocalDensity.current.density,
        fontScale = LocalDensity.current.fontScale
    )
    
    CompositionLocalProvider(LocalDensity provides customDensity) {
        content()
    }
}

// 方案 2：WindowSizeClass 响应式布局（推荐）
@Composable
fun AdaptiveLayout() {
    val windowSizeClass = calculateWindowSizeClass(this)
    
    when (windowSizeClass.widthSizeClass) {
        WindowWidthSizeClass.Compact -> {
            // 手机竖屏：单列布局
            PhoneLayout()
        }
        WindowWidthSizeClass.Medium -> {
            // 折叠屏/小平板：双栏布局
            TwoPaneLayout()
        }
        WindowWidthSizeClass.Expanded -> {
            // 大平板/横屏：三栏布局
            ThreePaneLayout()
        }
    }
}

// 方案 3：自定义尺寸系统
object AppDimens {
    val screenWidthDp: Dp
        @Composable get() = LocalConfiguration.current.screenWidthDp.dp
    
    // 按比例计算
    val cardWidth: Dp
        @Composable get() = screenWidthDp * 0.9f
    
    val horizontalPadding: Dp
        @Composable get() = screenWidthDp * 0.05f
}
```

**字体适配最佳实践：**

```kotlin
// 1. 不跟随系统字体缩放（适合固定布局）
textView.setTextSize(TypedValue.COMPLEX_UNIT_DIP, 14f) // 用 dp 代替 sp

// 2. 限制最大字体缩放（避免布局溢出）
override fun attachBaseContext(newBase: Context) {
    val config = Configuration(newBase.resources.configuration)
    config.fontScale = config.fontScale.coerceAtMost(1.3f) // 最大 1.3 倍
    super.attachBaseContext(newBase.createConfigurationContext(config))
}

// 3. Compose 中控制字体缩放
@Composable
fun NonScalableText(text: String, fontSize: TextUnit) {
    val density = LocalDensity.current
    val fixedFontSize = with(density) {
        // 将 sp 转为固定 dp，不受 fontScale 影响
        (fontSize.value / density.fontScale).sp
    }
    Text(text = text, fontSize = fixedFontSize)
}
```

**折叠屏适配：**

```kotlin
// 监听折叠状态
@Composable
fun FoldableAwareLayout() {
    val windowLayoutInfo = LocalWindowLayoutInfo.current
    val foldingFeature = windowLayoutInfo.displayFeatures
        .filterIsInstance<FoldingFeature>()
        .firstOrNull()
    
    when {
        foldingFeature?.state == FoldingFeature.State.HALF_OPENED -> {
            // 半折叠状态：上下分屏
            TableTopLayout(foldingFeature)
        }
        foldingFeature?.orientation == FoldingFeature.Orientation.VERTICAL -> {
            // 垂直折叠：左右分屏
            BookLayout(foldingFeature)
        }
        else -> {
            // 正常状态
            NormalLayout()
        }
    }
}
```

**实际项目选型建议：**

```
新项目（纯 Compose）→ WindowSizeClass + 自定义 Density
新项目（View 体系）→ smallestWidth 限定符
老项目快速适配 → 今日头条方案（AndroidAutoSize 库）
混合项目 → smallestWidth（View）+ CompositionLocal（Compose）
```
