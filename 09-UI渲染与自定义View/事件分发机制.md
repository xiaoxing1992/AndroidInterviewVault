# 事件分发机制

## 问题

"详细说一下 Android 事件分发机制，dispatchTouchEvent、onInterceptTouchEvent、onTouchEvent 三者的关系？事件序列是怎么处理的？滑动冲突怎么解决？"

## 核心答案（30秒版本）

事件分发遵循责任链模式：Activity → Window → DecorView → ViewGroup → View。核心三方法：dispatchTouchEvent 负责分发，onInterceptTouchEvent（仅 ViewGroup）决定是否拦截，onTouchEvent 负责消费。DOWN 事件确定消费者后，同一事件序列（MOVE/UP）直接交给该消费者。滑动冲突通过外部拦截法（父 onInterceptTouchEvent）或内部拦截法（子 requestDisallowInterceptTouchEvent）解决。

## 深入解析

### 原理层（源码级）

**事件分发伪代码模型：**

```java
// ViewGroup.dispatchTouchEvent 核心逻辑
public boolean dispatchTouchEvent(MotionEvent ev) {
    boolean consumed = false;
    
    if (onInterceptTouchEvent(ev)) {
        // 拦截：自己处理
        consumed = onTouchEvent(ev);
    } else {
        // 不拦截：交给子 View
        consumed = child.dispatchTouchEvent(ev);
    }
    
    // 子 View 不消费，自己兜底
    if (!consumed) {
        consumed = onTouchEvent(ev);
    }
    return consumed;
}
```

**ViewGroup.dispatchTouchEvent 源码关键逻辑：**

```java
// ViewGroup.java (API 34)
@Override
public boolean dispatchTouchEvent(MotionEvent ev) {
    boolean handled = false;
    final int action = ev.getAction();
    final int actionMasked = action & MotionEvent.ACTION_MASK;

    // 1. DOWN 事件重置状态
    if (actionMasked == MotionEvent.ACTION_DOWN) {
        cancelAndClearTouchTargets(ev);
        resetTouchState(); // 清除 FLAG_DISALLOW_INTERCEPT
    }

    // 2. 判断是否拦截
    final boolean intercepted;
    if (actionMasked == MotionEvent.ACTION_DOWN || mFirstTouchTarget != null) {
        // 子 View 可通过 requestDisallowInterceptTouchEvent 禁止父拦截
        final boolean disallowIntercept = (mGroupFlags & FLAG_DISALLOW_INTERCEPT) != 0;
        if (!disallowIntercept) {
            intercepted = onInterceptTouchEvent(ev);
        } else {
            intercepted = false;
        }
    } else {
        // 没有 touch target 且不是 DOWN，直接拦截
        intercepted = true;
    }

    // 3. DOWN 事件寻找消费者
    if (!intercepted && actionMasked == MotionEvent.ACTION_DOWN) {
        // 遍历子 View（按 Z 序从上到下）
        for (int i = childrenCount - 1; i >= 0; i--) {
            // 判断触摸点是否在子 View 范围内
            if (!child.canReceivePointerEvents() || 
                !isTransformedTouchPointInView(x, y, child, null)) {
                continue;
            }
            // 分发给子 View
            if (dispatchTransformedTouchEvent(ev, child)) {
                mFirstTouchTarget = addTouchTarget(child, idBitsToAssign);
                break;
            }
        }
    }

    // 4. 分发事件
    if (mFirstTouchTarget == null) {
        // 无消费者，自己处理（super.dispatchTouchEvent → onTouchEvent）
        handled = dispatchTransformedTouchEvent(ev, null);
    } else {
        // 分发给 touch target
        handled = dispatchTransformedTouchEvent(ev, mFirstTouchTarget.child);
    }
    return handled;
}
```

**事件序列规则：**

1. 一个事件序列：DOWN → MOVE...MOVE → UP（或 CANCEL）
2. DOWN 确定消费者后，后续 MOVE/UP 直接分发给该消费者，不再遍历子 View
3. 如果所有 View 都不消费 DOWN，事件回传给 Activity.onTouchEvent()
4. 父 ViewGroup 中途拦截 MOVE，子 View 收到 CANCEL，后续事件由父处理

**View.dispatchTouchEvent 源码：**

```java
// View.java
public boolean dispatchTouchEvent(MotionEvent event) {
    boolean result = false;
    // OnTouchListener 优先级高于 onTouchEvent
    if (mOnTouchListener != null && mOnTouchListener.onTouch(this, event)) {
        result = true;
    }
    if (!result && onTouchEvent(event)) {
        result = true;
    }
    return result;
}
```

**onTouchEvent 中的点击事件：**

```java
// View.java onTouchEvent
case MotionEvent.ACTION_UP:
    if (!mHasPerformedLongPress && !mIgnoreNextUpEvent) {
        // 这是一个点击
        if (mPerformClick == null) {
            mPerformClick = new PerformClick();
        }
        if (!post(mPerformClick)) {
            performClickInternal();
        }
    }
    break;
```

优先级链：OnTouchListener.onTouch() > onTouchEvent() > OnClickListener.onClick()

### 实战层（项目经验）

**外部拦截法（推荐）：**

```kotlin
// 父 ViewGroup 重写 onInterceptTouchEvent
class ConflictParentLayout : FrameLayout {
    private var lastX = 0f
    private var lastY = 0f

    override fun onInterceptTouchEvent(ev: MotionEvent): Boolean {
        when (ev.action) {
            MotionEvent.ACTION_DOWN -> {
                lastX = ev.x
                lastY = ev.y
                return false // DOWN 不拦截，否则子 View 收不到事件
            }
            MotionEvent.ACTION_MOVE -> {
                val dx = abs(ev.x - lastX)
                val dy = abs(ev.y - lastY)
                // 水平滑动距离 > 垂直滑动距离，父容器拦截
                return dx > dy && dx > ViewConfiguration.get(context).scaledTouchSlop
            }
            MotionEvent.ACTION_UP -> return false
        }
        return false
    }
}
```

**内部拦截法：**

```kotlin
// 子 View 重写 dispatchTouchEvent
class ConflictChildView : View {
    private var lastX = 0f
    private var lastY = 0f

    override fun dispatchTouchEvent(ev: MotionEvent): Boolean {
        when (ev.action) {
            MotionEvent.ACTION_DOWN -> {
                // 禁止父容器拦截
                parent.requestDisallowInterceptTouchEvent(true)
                lastX = ev.x
                lastY = ev.y
            }
            MotionEvent.ACTION_MOVE -> {
                val dx = abs(ev.x - lastX)
                val dy = abs(ev.y - lastY)
                if (dy > dx) {
                    // 垂直滑动，允许父容器拦截
                    parent.requestDisallowInterceptTouchEvent(false)
                }
            }
        }
        return super.dispatchTouchEvent(ev)
    }
}

// 父容器配合：onInterceptTouchEvent 中 DOWN 返回 false，其他返回 true
```

**常见滑动冲突场景：**

| 场景 | 解决方案 |
|---|---|
| ViewPager + ListView | 外部拦截：水平滑动父拦截 |
| ScrollView + RecyclerView | NestedScrollView 替代 ScrollView |
| CoordinatorLayout + AppBarLayout | Behavior 机制自动处理 |
| 下拉刷新 + 列表滚动 | 内部拦截：列表到顶时允许父拦截 |
