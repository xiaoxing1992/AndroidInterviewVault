# 代码混淆与反编译防护

## 问题

"R8 混淆的原理是什么？keep 规则怎么写？除了代码混淆还有哪些反编译防护手段？加固方案怎么选？"

## 核心答案（30秒版本）

R8 是 Android 官方的代码优化器，集成了混淆（重命名）、缩减（tree-shaking）、优化（内联/常量折叠）三大功能。通过 proguard-rules.pro 配置 keep 规则保护反射调用的类。防护体系还包括：资源混淆（AndResGuard）、SO 加固、DEX 加壳、反调试检测、完整性校验。商业加固方案（梆梆、360、腾讯乐固）提供 VMP 虚拟化保护和运行时防护。

## 深入解析

### 原理层（源码级）

**R8 处理流程：**

```
.java/.kt 源码
    ↓ (javac/kotlinc)
.class 字节码
    ↓ (D8/R8)
┌─────────────────────────────────────┐
│  R8 优化管线：                        │
│  1. Tree Shaking — 移除未使用的类/方法 │
│  2. Optimization — 内联、常量传播      │
│  3. Obfuscation — 类名/方法名重命名    │
│  4. Code Shrinking — 移除死代码       │
└─────────────────────────────────────┘
    ↓
.dex (优化后的 Dalvik 字节码)
```

**R8 混淆重命名策略：**

```
// 混淆前
com.example.app.network.UserApiService.getUserProfile(String userId)

// 混淆后
a.b.c.d.a(String a)

// mapping.txt 记录映射关系（用于崩溃日志还原）
com.example.app.network.UserApiService -> a.b.c.d:
    java.lang.String getUserProfile(java.lang.String) -> a
```

**R8 与 ProGuard 的区别：**

| 特性 | ProGuard | R8 |
|------|----------|-----|
| 维护者 | GuardSquare | Google |
| 集成方式 | 独立工具 | AGP 内置 |
| 输入 | .class → .class | .class → .dex（一步到位） |
| Kotlin 支持 | 有限 | 原生支持 metadata |
| 优化能力 | 基础 | 更激进（内联、枚举优化） |
| 兼容性 | ProGuard 规则 | 兼容 ProGuard 规则 + R8 专属规则 |

### 实战层（项目经验）

**build.gradle.kts 配置：**

```kotlin
android {
    buildTypes {
        release {
            isMinifyEnabled = true      // 启用代码缩减和混淆
            isShrinkResources = true    // 启用资源缩减
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
        }
    }
}
```

**常用 keep 规则模板：**

```proguard
# ===== 基础规则 =====

# 保留四大组件（AndroidManifest 注册的类）
-keep public class * extends android.app.Activity
-keep public class * extends android.app.Service
-keep public class * extends android.content.BroadcastReceiver
-keep public class * extends android.content.ContentProvider

# ===== 反射相关 =====

# 保留 Gson 序列化的 data class
-keep class com.example.app.model.** { *; }
# 或使用 @Keep 注解
-keep @androidx.annotation.Keep class * { *; }

# 保留枚举（反射访问 values() 和 valueOf()）
-keepclassmembers enum * {
    public static **[] values();
    public static ** valueOf(java.lang.String);
}

# ===== Retrofit 规则 =====

# 保留 Retrofit 接口方法签名（注解 + 参数类型）
-keepattributes Signature
-keepattributes *Annotation*
-keep,allowobfuscation interface * {
    @retrofit2.http.* <methods>;
}

# ===== Kotlin 相关 =====

# 保留 Kotlin Metadata（反射需要）
-keep class kotlin.Metadata { *; }
# 保留协程相关
-keepclassmembers class kotlinx.coroutines.** { *; }

# ===== JNI 相关 =====

# 保留 native 方法名（JNI 通过名称查找）
-keepclasseswithmembernames class * {
    native <methods>;
}

# ===== WebView JavaScript 接口 =====

# 保留 @JavascriptInterface 注解的方法
-keepclassmembers class * {
    @android.webkit.JavascriptInterface <methods>;
}
```

**资源混淆（AndResGuard）：**

```kotlin
// build.gradle.kts
andResGuard {
    mappingFile = file("resource_mapping.txt")
    use7zip = true
    useSign = true
    keepRoot = false
    // 白名单：不混淆的资源
    whiteList = listOf(
        "R.drawable.icon",
        "R.string.app_name",
        "R.mipmap.*"
    )
    // 压缩白名单：不压缩的文件类型
    compressFilePattern = listOf(
        "*.png", "*.jpg", "*.gif"
    )
}

// 效果：res/drawable/background_main.png → res/a/b.png
// APK 体积减少 1-3MB
```

**反调试检测：**

```kotlin
object AntiDebug {
    // 检测调试器连接
    fun isDebuggerAttached(): Boolean {
        return Debug.isDebuggerConnected() || Debug.waitingForDebugger()
    }
    
    // 检测 TracerPid（ptrace 附加）
    fun isBeingTraced(): Boolean {
        val statusFile = File("/proc/self/status")
        val tracerPid = statusFile.readLines()
            .find { it.startsWith("TracerPid:") }
            ?.split(":")?.get(1)?.trim()?.toIntOrNull()
        return tracerPid != null && tracerPid > 0
    }
    
    // 检测 Frida
    fun isFridaDetected(): Boolean {
        // 检查 Frida 默认端口
        return try {
            Socket("127.0.0.1", 27042).close()
            true
        } catch (e: Exception) {
            false
        }
    }
    
    // 检测 root
    fun isRooted(): Boolean {
        val paths = arrayOf("/system/bin/su", "/system/xbin/su", "/sbin/su")
        return paths.any { File(it).exists() }
    }
    
    // 签名校验（防止重打包）
    fun verifySignature(context: Context, expectedHash: String): Boolean {
        val packageInfo = context.packageManager.getPackageInfo(
            context.packageName, PackageManager.GET_SIGNING_CERTIFICATES
        )
        val signatures = packageInfo.signingInfo.apkContentsSigners
        val certHash = MessageDigest.getInstance("SHA-256")
            .digest(signatures[0].toByteArray())
        return Base64.encodeToString(certHash, Base64.NO_WRAP) == expectedHash
    }
}
```

**完整性校验（DEX 校验）：**

```kotlin
// 校验 classes.dex 的 CRC 值，防止被修改
fun verifyDexIntegrity(context: Context, expectedCrc: Long): Boolean {
    val apkPath = context.packageCodePath
    val zipFile = ZipFile(apkPath)
    val dexEntry = zipFile.getEntry("classes.dex")
    return dexEntry.crc == expectedCrc
}
```

**商业加固方案对比：**

| 方案 | VMP | DEX 加壳 | SO 保护 | 反调试 | 环境检测 |
|------|-----|---------|---------|--------|---------|
| 梆梆安全 | 支持 | 支持 | 支持 | 支持 | 支持 |
| 360 加固 | 支持 | 支持 | 支持 | 支持 | 支持 |
| 腾讯乐固 | 支持 | 支持 | 支持 | 支持 | 支持 |
| DexGuard | 支持 | 支持 | 有限 | 支持 | 有限 |

**加固对性能的影响：**

- 冷启动时间增加 100-500ms（DEX 解壳）
- 内存占用增加 5-15MB
- 方法调用开销增加（VMP 虚拟化方法）
- 建议：核心逻辑加固，非核心代码仅混淆

## 记忆锚点

R8 三合一：缩减 + 优化 + 混淆；keep 规则保护反射入口；防护层次：混淆（基础）→ 加壳（中级）→ VMP 虚拟化（高级）→ 运行时检测（动态）。
