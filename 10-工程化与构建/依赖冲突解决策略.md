# 依赖冲突解决策略

## 问题

> Gradle 中依赖冲突是如何产生的？默认的解决策略是什么？如何排查和解决版本冲突？force、exclude、strictly 各有什么用？

## 核心答案

依赖冲突产生于多个模块依赖同一库的不同版本。Gradle 默认策略是选择最高版本（newest wins）。排查用 `dependencies` 任务查看依赖树，`→` 标记表示版本被升级。解决手段：`force` 强制指定版本、`exclude` 排除传递依赖、`strictly` 严格锁定版本（不允许被升级）、Resolution Strategy 全局控制。

## 深入解析

### 原理层

**冲突产生场景：**
```
app 依赖:
├── library-a:1.0 → 依赖 okhttp:4.9.0
└── library-b:2.0 → 依赖 okhttp:4.11.0

冲突: okhttp 4.9.0 vs 4.11.0
Gradle 默认: 选择 4.11.0（最高版本）
```

**依赖树分析：**
```bash
./gradlew :app:dependencies --configuration runtimeClasspath

# 输出示例:
+--- com.example:library-a:1.0
|    \--- com.squareup.okhttp3:okhttp:4.9.0 -> 4.11.0  # → 表示被升级
\--- com.example:library-b:2.0
     \--- com.squareup.okhttp3:okhttp:4.11.0
```

**解决手段：**

```kotlin
// 1. force — 强制使用指定版本
configurations.all {
    resolutionStrategy {
        force("com.squareup.okhttp3:okhttp:4.10.0")
    }
}

// 2. exclude — 排除传递依赖
implementation("com.example:library-a:1.0") {
    exclude(group = "com.squareup.okhttp3", module = "okhttp")
}

// 3. strictly — 严格版本约束（不允许被其他依赖升级）
implementation("com.squareup.okhttp3:okhttp") {
    version { strictly("4.10.0") }
}

// 4. Resolution Strategy — 全局策略
configurations.all {
    resolutionStrategy {
        // 遇到冲突直接报错（而非静默选择最高版本）
        failOnVersionConflict()
        
        // 强制统一版本
        eachDependency {
            if (requested.group == "org.jetbrains.kotlin") {
                useVersion("1.9.22")
            }
        }
    }
}
```

**BOM（Bill of Materials）统一版本：**
```kotlin
// 使用 BOM 统一一组库的版本
implementation(platform("androidx.compose:compose-bom:2024.02.00"))
implementation("androidx.compose.ui:ui")           // 版本由 BOM 决定
implementation("androidx.compose.material3:material3")  // 版本由 BOM 决定

// OkHttp BOM
implementation(platform("com.squareup.okhttp3:okhttp-bom:4.11.0"))
implementation("com.squareup.okhttp3:okhttp")
implementation("com.squareup.okhttp3:logging-interceptor")
```

**Version Catalog 统一管理：**
```toml
# gradle/libs.versions.toml
[versions]
okhttp = "4.11.0"
kotlin = "1.9.22"

[libraries]
okhttp = { module = "com.squareup.okhttp3:okhttp", version.ref = "okhttp" }
okhttp-logging = { module = "com.squareup.okhttp3:logging-interceptor", version.ref = "okhttp" }
```

### 实战层

**排查冲突的工具：**
```bash
# 查看特定依赖的版本选择原因
./gradlew :app:dependencyInsight --dependency okhttp --configuration runtimeClasspath

# 输出:
# com.squareup.okhttp3:okhttp:4.11.0
#    variant "runtime" [
#       ...
#    ]
#    Selection reasons:
#       - By conflict resolution: between versions 4.11.0 and 4.9.0
```

**常见冲突场景及解决：**
```kotlin
// Kotlin 版本冲突（最常见）
configurations.all {
    resolutionStrategy.eachDependency {
        if (requested.group == "org.jetbrains.kotlin") {
            useVersion(libs.versions.kotlin.get())
        }
    }
}

// AndroidX 与 Support Library 冲突
// 使用 Jetifier 自动转换
android.enableJetifier=true  // gradle.properties

// 重复类冲突（Duplicate class）
// 通常需要 exclude 其中一个
implementation("com.example:lib") {
    exclude(group = "com.google.code.gson")  // 排除内嵌的 gson
}
```

**依赖锁定（Dependency Locking）：**
```kotlin
// 锁定所有依赖版本，防止意外升级
dependencyLocking {
    lockAllConfigurations()
}
// 生成: gradle/dependency-locks/*.lockfile
// 更新: ./gradlew dependencies --write-locks
```

## 记忆锚点

Gradle 默认选最高版本。排查用 `dependencyInsight`，解决用 force（强制）/ exclude（排除）/ strictly（锁死）/ BOM（统一一族）。Version Catalog 是版本管理的最佳实践。
