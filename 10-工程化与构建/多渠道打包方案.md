# 多渠道打包方案

## 问题

> Android 多渠道打包有哪些方案？Gradle productFlavors 的原理是什么？如何实现快速打包几百个渠道包？

## 核心答案

三种主流方案：Gradle productFlavors（官方，每个渠道独立编译，慢但灵活）、V2 签名方案写入渠道信息（美团 Walle，秒级打包）、META-INF 写入空文件（腾讯 VasDolly，兼容 V1 签名）。大量渠道包推荐 Walle/VasDolly 方案，在已签名 APK 的 APK Signing Block 中写入渠道信息，无需重新编译。

## 深入解析

### 原理层

**方案1: Gradle productFlavors（官方）：**
```kotlin
android {
    flavorDimensions += "channel"
    productFlavors {
        create("googleplay") { dimension = "channel" }
        create("huawei") { dimension = "channel" }
        create("xiaomi") { dimension = "channel" }
    }
}
// 每个 flavor 独立编译 → N 个渠道 = N 次编译
// 优点：每个渠道可以有不同代码/资源
// 缺点：100 个渠道 = 100 次编译，极慢
```

**方案2: APK Signing Block 写入（Walle）：**
```
APK 文件结构:
┌─────────────────────┐
│ ZIP 内容            │
├─────────────────────┤
│ APK Signing Block   │ ← 在这里写入渠道信息
│ (V2/V3 签名区域)    │    不影响签名校验
├─────────────────────┤
│ Central Directory   │
├─────────────────────┤
│ End of Central Dir  │
└─────────────────────┘

原理: V2 签名只校验 ZIP 内容和 Central Directory
     APK Signing Block 中的自定义 ID-Value 对不参与校验
     → 写入渠道信息不破坏签名
```

```kotlin
// Walle 使用
// 1. 正常签名打包一个 APK
// 2. 用 Walle 工具写入渠道信息
// walle put -c xiaomi app-release.apk → app-release-xiaomi.apk

// 读取渠道
val channel = WalleChannelReader.getChannel(context)
```

**方案3: META-INF 空文件（VasDolly）：**
```
原理: V1 签名不校验 META-INF 目录下新增的文件
     在 META-INF/ 下添加 channel_xiaomi 空文件
     → V1 签名仍然有效

// 读取渠道
val channel = zipFile.entries().find { it.name.startsWith("META-INF/channel_") }
    ?.name?.removePrefix("META-INF/channel_")
```

**方案对比：**

| 方案 | 速度 | V1 签名 | V2 签名 | 灵活性 |
|------|------|---------|---------|--------|
| productFlavors | 慢（N次编译） | ✅ | ✅ | 高（可改代码） |
| Walle | 秒级 | ❌ | ✅ | 低（只能写渠道） |
| VasDolly | 秒级 | ✅ | ✅ | 低 |
| productFlavors + 渠道包 | 中 | ✅ | ✅ | 中 |

### 实战层

**Walle Gradle 插件配置：**
```kotlin
// build.gradle.kts
plugins {
    id("com.meituan.android.walle")
}

walle {
    channelFile = file("channel.txt")  // 渠道列表文件
    // xiaomi
    // huawei
    // oppo
    // vivo
    // ...
}

// 执行: ./gradlew assembleReleaseChannels
// 输出: build/outputs/apk/release/ 下生成所有渠道包
```

**动态渠道信息：**
```kotlin
// 除了渠道名，还可以写入额外信息
// Walle 支持写入 JSON
val extraInfo = mapOf("channel" to "xiaomi", "timestamp" to System.currentTimeMillis())
```

**选型建议：**
- 渠道 < 10 个 + 需要不同代码/资源 → productFlavors
- 渠道 > 10 个 + 只需要渠道标识 → Walle（V2 签名）
- 需要兼容 V1 签名 → VasDolly
- Google Play 单渠道 → 不需要多渠道方案

## 记忆锚点

多渠道打包：少量渠道用 Gradle Flavor（慢但灵活），大量渠道用 Walle（在签名区域塞渠道信息，秒级出包，不破坏签名）。
