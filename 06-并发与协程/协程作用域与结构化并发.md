# 协程作用域与结构化并发

## 问题

"说说你对结构化并发的理解？CoroutineScope、supervisorScope、viewModelScope 各自的设计意图是什么？子协程异常时父协程会怎样？"

## 核心答案（30秒版本）

结构化并发的核心原则是：协程的生命周期被限定在一个明确的作用域内，父协程负责管理所有子协程，子协程异常会向上传播取消整个作用域（除非使用 SupervisorJob）。viewModelScope 绑定 ViewModel 生命周期自动取消，supervisorScope 隔离子协程失败不影响兄弟协程。这避免了协程泄漏和"fire-and-forget"问题。

## 深入解析

### 原理层（源码级）

**CoroutineScope 的本质：**

```kotlin
// CoroutineScope 只是一个接口，持有 CoroutineContext
public interface CoroutineScope {
    public val coroutineContext: CoroutineContext
}

// 关键：Context 中必须包含 Job，Job 构成父子树
public fun CoroutineScope(context: CoroutineContext): CoroutineScope =
    ContextScope(if (context[Job] != null) context else context + Job())
```

**父子关系建立（源码追踪）：**

```kotlin
// launch 内部调用链：
public fun CoroutineScope.launch(...): Job {
    val newContext = newCoroutineContext(context)  // 继承父 context
    val coroutine = StandaloneCoroutine(newContext, active = true)
    coroutine.start(start, coroutine, block)
    return coroutine
}

// AbstractCoroutine.initParentJob():
internal fun initParentJob(parent: Job?) {
    if (parent == null) return
    // 关键：将自己注册为父 Job 的子节点
    val handle = parent.attachChild(this)
    parentHandle = handle
}
```

**异常传播机制：**

```kotlin
// JobSupport.childCancelled() — 普通 Job
// 子协程异常 → 调用父 Job 的 childCancelled → 取消父 Job → 取消所有子 Job
override fun childCancelled(cause: Throwable): Boolean {
    if (cause is CancellationException) return true  // CancellationException 不传播
    return cancelImpl(cause)  // 其他异常向上传播
}

// SupervisorJob 重写了这个方法：
override fun childCancelled(cause: Throwable): Boolean = false  // 不传播！
```

**coroutineScope vs supervisorScope：**

```kotlin
// coroutineScope：任一子协程失败 → 取消所有兄弟 → 向上抛异常
suspend fun loadData() = coroutineScope {
    val user = async { fetchUser() }      // 失败会取消 orders
    val orders = async { fetchOrders() }  // 失败会取消 user
    Pair(user.await(), orders.await())
}

// supervisorScope：子协程独立，失败不影响兄弟
suspend fun loadDataSafe() = supervisorScope {
    val user = async { fetchUser() }      // 失败不影响 orders
    val orders = async { fetchOrders() }  // 失败不影响 user
    // 但需要自己处理异常！
    val u = runCatching { user.await() }.getOrNull()
    val o = runCatching { orders.await() }.getOrNull()
    Pair(u, o)
}
```

### 实战层（项目经验）

**常见错误 1：GlobalScope 滥用**

```kotlin
// 错误：协程泄漏，Activity 销毁后仍在执行
class MyActivity : AppCompatActivity() {
    fun loadData() {
        GlobalScope.launch {  // ❌ 不受生命周期管理
            val data = api.fetch()
            updateUI(data)  // 可能 crash：Activity 已销毁
        }
    }
}

// 正确：使用 lifecycleScope
class MyActivity : AppCompatActivity() {
    fun loadData() {
        lifecycleScope.launch {  // ✅ Activity 销毁自动取消
            val data = api.fetch()
            updateUI(data)
        }
    }
}
```

**常见错误 2：supervisorScope 误用**

```kotlin
// 错误：以为 launch 里加 SupervisorJob 就能隔离
viewModelScope.launch(SupervisorJob()) {  // ❌ 断开了父子关系！
    // 这个协程不再受 viewModelScope 管理
}

// 正确：使用 supervisorScope 函数
viewModelScope.launch {
    supervisorScope {  // ✅ 保持父子关系，但子协程独立
        launch { task1() }
        launch { task2() }
    }
}
```

**实战模式：并行请求 + 部分失败容忍**

```kotlin
class HomeViewModel : ViewModel() {
    fun loadHomePage() {
        viewModelScope.launch {
            // Banner 失败不影响其他模块
            supervisorScope {
                val banner = async { bannerRepo.fetch() }
                val recommend = async { recommendRepo.fetch() }
                val activity = async { activityRepo.fetch() }

                _state.update { state ->
                    state.copy(
                        banner = banner.awaitOrNull(),
                        recommend = recommend.awaitOrNull(),
                        activity = activity.awaitOrNull()
                    )
                }
            }
        }
    }

    private suspend fun <T> Deferred<T>.awaitOrNull(): T? =
        runCatching { await() }.getOrNull()
}
```

**实战模式：自定义 Scope 管理后台任务**

```kotlin
// Application 级别的长生命周期 Scope
class AppCoroutineScope @Inject constructor() : CoroutineScope {
    override val coroutineContext = SupervisorJob() + Dispatchers.Default +
        CoroutineExceptionHandler { _, throwable ->
            CrashReporter.report(throwable)
        }

    fun destroy() {
        coroutineContext.cancelChildren()
    }
}

// 使用场景：埋点上报、日志写入等不依赖 UI 生命周期的任务
```

## 记忆锚点

结构化并发就像"公司组织架构"——员工（子协程）归属部门（Scope），部门撤销则员工全部离职，SupervisorJob 是"独立子公司"模式，一个子公司倒闭不影响其他子公司。
