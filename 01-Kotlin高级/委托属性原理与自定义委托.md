# 委托属性原理与自定义委托

## 问题

> Kotlin 委托属性的底层原理是什么？by lazy 是如何实现的？如何自定义一个属性委托？

## 核心答案

委托属性通过 `by` 关键字将属性的 getter/setter 委托给一个对象，编译器生成一个隐藏的委托字段，属性访问被转发为委托对象的 `getValue`/`setValue` 调用。`by lazy` 内部使用 `SynchronizedLazyImpl`，通过双重检查锁实现线程安全的延迟初始化。

## 深入解析

### 原理层

**编译后的代码：**
```kotlin
// 源码
class Example {
    var name: String by Delegates.observable("init") { _, old, new ->
        println("$old -> $new")
    }
}

// 编译后等价于
class Example {
    private val name$delegate = Delegates.observable("init") { _, old, new ->
        println("$old -> $new")
    }
    var name: String
        get() = name$delegate.getValue(this, ::name)
        set(value) { name$delegate.setValue(this, ::name, value) }
}
```

**by lazy 实现：**
```kotlin
// LazyThreadSafetyMode.SYNCHRONIZED（默认）
private class SynchronizedLazyImpl<out T>(initializer: () -> T) : Lazy<T> {
    private var initializer: (() -> T)? = initializer
    @Volatile private var _value: Any? = UNINITIALIZED_VALUE

    override val value: T
        get() {
            if (_value !== UNINITIALIZED_VALUE) return _value as T
            synchronized(this) {
                if (_value !== UNINITIALIZED_VALUE) return _value as T
                val typedValue = initializer!!()
                _value = typedValue
                initializer = null  // 释放引用，帮助 GC
                return typedValue
            }
        }
}
```

三种线程安全模式：
- `SYNCHRONIZED` — 双重检查锁（默认，最安全）
- `PUBLICATION` — CAS，允许多次初始化但只取第一个结果
- `NONE` — 无同步，单线程场景使用

**自定义委托接口：**
```kotlin
operator fun getValue(thisRef: R, property: KProperty<*>): T
operator fun setValue(thisRef: R, property: KProperty<*>, value: T)
```

### 实战层

**自定义 SharedPreferences 委托：**
```kotlin
class PreferenceDelegate<T>(
    private val prefs: SharedPreferences,
    private val key: String,
    private val default: T
) : ReadWriteProperty<Any?, T> {
    override fun getValue(thisRef: Any?, property: KProperty<*>): T {
        return prefs.get(key, default)
    }
    override fun setValue(thisRef: Any?, property: KProperty<*>, value: T) {
        prefs.edit { put(key, value) }
    }
}

// 使用
var token: String by PreferenceDelegate(prefs, "token", "")
```

**常见内置委托：**
- `by lazy` — 延迟初始化
- `by Delegates.observable` — 属性变化监听
- `by Delegates.vetoable` — 属性变化拦截
- `by map` — 从 Map 中读取属性

**坑点：**
- `by lazy` 在 Android 中如果持有 Activity 引用可能导致内存泄漏
- `by viewModels()` 是 Fragment KTX 提供的委托，内部使用 ViewModelProvider
