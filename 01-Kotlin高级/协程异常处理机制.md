# 协程异常处理机制

## 问题

> Kotlin 协程中异常是如何传播的？CoroutineExceptionHandler 在什么位置设置才有效？supervisorScope 和 coroutineScope 的异常行为有什么区别？

## 核心答案

协程异常默认向上传播（取消父协程及所有兄弟协程）。CoroutineExceptionHandler 只在根协程（顶层 launch）中生效，是最后的兜底处理器。supervisorScope 改变传播方向：子协程失败不影响父和兄弟，异常由子协程自己处理。coroutineScope 中任一子协程失败会取消整个作用域。

## 深入解析

### 原理层

**异常传播机制：**
```
launch(根) → launch(子A) → 异常！
                              ↓
                         通知父协程
                              ↓
                    父协程取消所有子协程（包括子B）
                              ↓
                    父协程自身也被取消
                              ↓
                    CoroutineExceptionHandler 处理
```

**关键规则：**
1. `launch` 产生的异常会传播（fire-and-forget 语义）
2. `async` 产生的异常在 `await()` 时抛出（deferred 语义）
3. `CoroutineExceptionHandler` 只对 `launch` 有效，对 `async` 无效

**CoroutineExceptionHandler 生效位置：**
```kotlin
// ✅ 有效 — 设置在根协程
val handler = CoroutineExceptionHandler { _, e -> log(e) }
scope.launch(handler) {
    launch { throw RuntimeException() } // handler 会捕获
}

// ❌ 无效 — 设置在子协程
scope.launch {
    launch(handler) { throw RuntimeException() } // handler 不会被调用
}
```

**supervisorScope vs coroutineScope：**
```kotlin
// coroutineScope — 一个失败，全部取消
coroutineScope {
    launch { delay(1000); println("不会执行") }
    launch { throw Exception() }  // 取消整个 scope
}

// supervisorScope — 互不影响
supervisorScope {
    launch { delay(1000); println("正常执行") }
    launch { throw Exception() }  // 只影响自己
}
```

**底层实现：**
- `JobSupport.childCancelled()` — 普通 Job 返回 true（接受子异常），SupervisorJob 返回 false（拒绝）
- 异常传播链：`child.cancel()` → `parent.childCancelled()` → `parent.cancelImpl()` → 取消所有 children
- `handleCoroutineException()` — 最终调用 `CoroutineExceptionHandler` 或 Thread.uncaughtExceptionHandler

### 实战层

**Android ViewModel 中的最佳实践：**
```kotlin
class MyViewModel : ViewModel() {
    // viewModelScope 使用 SupervisorJob，子协程互不影响
    
    fun loadData() {
        viewModelScope.launch {
            try {
                val result = repository.fetchData()
                _state.value = Success(result)
            } catch (e: Exception) {
                _state.value = Error(e)
            }
        }
    }
}
```

**常见错误模式：**
```kotlin
// ❌ 错误：try-catch 无法捕获子协程的异常
scope.launch {
    try {
        launch { throw Exception() }  // 异常绕过 try-catch 向上传播
    } catch (e: Exception) {
        // 捕获不到！
    }
}

// ✅ 正确：用 coroutineScope 或在子协程内部 try-catch
scope.launch {
    coroutineScope {
        launch {
            try { riskyOperation() } catch (e: Exception) { handle(e) }
        }
    }
}
```

**CancellationException 特殊性：**
- 不会传播给父协程（正常取消流程）
- 不会被 CoroutineExceptionHandler 处理
- `try-catch` 捕获后不要吞掉，否则协程无法正常取消
